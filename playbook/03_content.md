# 03 - Content Preparation (4-2 Weeks Out)

## Objective
Prepare all workshop content, tools, and materials so the day runs smoothly.

## The Workshop Formula

This hackathon uses two AWS frameworks:
1. **Working Backwards** (business strategy methodology)
2. **PartyRock** (no-code GenAI app builder)

Students learn WB methodology, then build a GenAI app using PartyRock, then pitch it.

## Step-by-Step

### 1. PartyRock Setup
- [ ] Go to https://partyrock.aws
- [ ] Create an event token (contact your AWS event coordinator)
- [ ] Test the workshop guide link works
- [ ] Prepare a demo app (like "Meet Whiskers" or similar)
- [ ] Verify PartyRock works on university WiFi (test if possible)

### 2. Presentation Deck
- [ ] Use or adapt the main presentation template
- [ ] Key sections to include:
  - Welcome and agenda
  - AWS Working Backwards overview (15-20 min)
  - PartyRock demo and hands-on instructions
  - Evaluation rubric (show students what they are being judged on)
  - Presentation prep guidelines
  - Judge bios
  - AWS learning resources
- [ ] Customize judge bios for your team
- [ ] Add university-specific branding if desired

### 3. Student Deliverables (What They Build)
Each team produces:
1. Professional PR/FAQ Document (Product Strategy)
2. Working AI Application in PartyRock (Technical Implementation)
3. Image Generation Portfolio (Visual Design)
4. Technical Documentation (Professional Writing)
5. Product Pitch (7 min presentation)

### 4. Team Formation Plan
- [ ] Recommended: let students self-organize into teams of 3-5
- [ ] Have a backup plan for stragglers (assign them to smaller teams)
- [ ] Target: 8-12 teams total

## Deliverables
- [ ] Presentation deck finalized
- [ ] PartyRock event code working
- [ ] Workshop guide link tested
- [ ] Team formation plan decided

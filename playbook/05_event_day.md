# 05 - Event Day Run of Show

## Before Students Arrive (9:00 - 10:00 AM)

- [ ] Arrive 1 hour early
- [ ] Set up AV (test projector, mic, screen sharing)
- [ ] Set up sign-in table
- [ ] Arrange tables for teams (groups of 4-6)
- [ ] Put team number cards on tables
- [ ] Test WiFi (connect multiple devices)
- [ ] Test PartyRock access on university WiFi
- [ ] Set up catering area
- [ ] Brief volunteers one final time (5 min huddle)
- [ ] Open scoring app, confirm it works

## Opening (10:00 - 10:15 AM)

- [ ] Welcome students, direct to sign-in
- [ ] Event Lead: opening remarks (5 min)
  - Welcome, thank university partner
  - Introduce AWS team briefly
  - "By 3pm you will have a portfolio-ready GenAI app"

## Working Backwards Presentation (10:15 - 10:45 AM)

- [ ] Presenter delivers WB methodology (20 min)
- [ ] Show sample PR/FAQ (5 min)
- [ ] Q&A (5 min)

## Hands-On Workshop (10:45 AM - 12:00 PM)

- [ ] PartyRock presenter: demo the tool (10 min)
- [ ] Share event code and workshop link
- [ ] Students form teams and start building
- [ ] Mentors circulate, help teams
- [ ] Check in on teams at 11:15 and 11:45

## Lunch + Career Panel (12:00 - 1:00 PM)

- [ ] Announce lunch
- [ ] Career panel at 12:15 (2-3 speakers, 5 min each, then Q&A)
- [ ] Students can continue working during lunch

## Presentation Prep (1:00 - 2:00 PM)

- [ ] Announce: "Presentations start at 2:15. You have 1 hour to finalize."
- [ ] Segment Lead: collect team names, assign presentation order
- [ ] Set up scoring app with team names

## Judging (2:15 - 3:00 PM)

- [ ] Segment Lead announces presentation order
- [ ] Timer: strict 7 minutes per team
- [ ] Judges score in real-time using the app
- [ ] After all presentations: judges confer briefly (5 min)
- [ ] Announce winners (top 3)

## Closing (Last 10 min)

- [ ] Thank students, university, volunteers
- [ ] Share event survey link
- [ ] Share AWS learning resources
- [ ] Group photo

## Troubleshooting

| Problem | Solution |
|---------|----------|
| WiFi issues | Mobile hotspot as backup |
| PartyRock down | Pivot to conceptual presentations only |
| Low attendance | Combine smaller teams, adjust timeline |
| AV failure | Use laptop screen for small groups |
| Running behind | Cut lunch panel short, reduce Q&A time |
